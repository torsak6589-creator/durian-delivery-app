# คู่มือติดตั้ง — ระบบบันทึกการจัดส่งทุเรียน (ฐานข้อมูลกลาง)

ระบบนี้ใช้ **Supabase** เป็นฐานข้อมูลกลาง ทุกคนที่เปิดลิงก์เดียวกันจะเห็นและแก้ไขข้อมูลชุดเดียวกัน รองรับ realtime (เห็นการเปลี่ยนแปลงทันที) และทำงานต่อได้แม้เน็ตหลุด (บันทึกในเครื่องแล้วซิงค์ทีหลัง)

---

## ขั้นที่ 1 — สร้างโปรเจกต์ Supabase (ฟรี)

1. ไปที่ <https://supabase.com> → กด **Start your project** → สมัคร/เข้าสู่ระบบ (ใช้บัญชี GitHub หรืออีเมลบริษัทก็ได้)
2. กด **New project**
   - ตั้งชื่อ เช่น `durian-ff`
   - ตั้ง **Database Password** (เก็บไว้ให้ดี)
   - เลือก Region: **Southeast Asia (Singapore)** (ใกล้ไทยที่สุด เร็วกว่า)
3. รอสร้างเสร็จประมาณ 1–2 นาที

---

## ขั้นที่ 2 — สร้างตารางในฐานข้อมูล

1. ในโปรเจกต์ Supabase → เมนูซ้าย **SQL Editor** → **New query**
2. เปิดไฟล์ **`supabase-schema.sql`** (อยู่ในโฟลเดอร์เดียวกันนี้) คัดลอกทั้งหมดมาวาง
3. กดปุ่ม **Run** (มุมขวาล่าง)
4. ถ้าขึ้น `Success` แปลว่าเรียบร้อย ✓

---

## ขั้นที่ 3 — เอาค่าเชื่อมต่อมาใส่ในแอป

1. ในโปรเจกต์ Supabase → **Settings** (เฟือง) → **API**
2. คัดลอก 2 ค่านี้:
   - **Project URL** — เช่น `https://abcdxyz.supabase.co`
   - **anon public** key — ข้อความยาว ๆ ขึ้นต้นด้วย `eyJ...`

### มี 2 วิธีใส่ค่า — เลือกอย่างใดอย่างหนึ่ง

**วิธี A (แนะนำสำหรับ deploy จริง):** ฝังค่าไว้ในไฟล์เลย ทุกคนที่เปิดลิงก์ไม่ต้องกรอกเอง
- เปิดไฟล์ **`db.js`** → หาบรรทัด `const HARDCODED = {` ด้านบนสุด
- ใส่ค่าลงไป:
  ```js
  const HARDCODED = {
    url: "https://abcdxyz.supabase.co",
    key: "eyJhbGciOiJIUzI1NiIs....(anon key ของคุณ)",
  };
  ```
- บันทึกไฟล์

**วิธี B (ทดสอบเร็ว):** ไม่ต้องแก้ไฟล์ — เปิดแอป แล้วหน้าตั้งค่าจะเด้งขึ้นมาให้กรอก URL + key (เก็บไว้ในเครื่องนั้นเครื่องเดียว)

> 🔒 anon key ปลอดภัยที่จะเปิดเผยในไฟล์ เพราะถูกควบคุมด้วย Row Level Security ที่เราตั้งไว้แล้ว

---

## ขั้นที่ 4 — Deploy ขึ้น GitHub Pages

1. สร้าง repository ใหม่บน GitHub (เช่น `durian-ff`)
2. อัปโหลดไฟล์ทั้งหมดในโฟลเดอร์ `app/` ขึ้นไป:
   - `index.html`
   - `helpers.js`
   - `db.js`
   - `app.jsx`
   - (ไฟล์ `supabase-schema.sql` และ README จะอัปด้วยก็ได้ ไม่กระทบการทำงาน)
3. ไปที่ repository → **Settings** → **Pages**
   - **Source**: เลือก `Deploy from a branch`
   - **Branch**: เลือก `main` / โฟลเดอร์ `/ (root)` → **Save**
4. รอสักครู่ GitHub จะให้ลิงก์ เช่น `https://ชื่อคุณ.github.io/durian-ff/`
5. ส่งลิงก์นี้ให้ทุกคนในทีม — เปิดได้ทั้งมือถือและคอม ใช้ฐานข้อมูลเดียวกัน

> หมายเหตุ: ถ้าใช้ **วิธี B** (ไม่ฝังค่าใน `db.js`) แต่ละเครื่องต้องกรอก URL + key เองครั้งแรก ดังนั้นสำหรับใช้งานจริงทั้งทีม **แนะนำวิธี A**

---

## ย้ายข้อมูลเดิม (ที่เคยบันทึกในเครื่อง)

- ถ้าเครื่องไหนเคยใช้แอปเวอร์ชันเดิมและมีข้อมูลค้างอยู่ เมื่อเปิดแอปใหม่ที่เชื่อมฐานข้อมูลแล้ว จะมีแถบสีน้ำเงินขึ้นว่า **"พบข้อมูลเดิมในเครื่องนี้ N รายการ"**
- กด **"ย้ายขึ้นคลาวด์"** ข้อมูลเดิมจะถูกอัปขึ้นฐานข้อมูลกลางให้ทุกคนเห็น
- หรือใช้ปุ่ม **นำเข้า Excel** ในหน้าประวัติ เพื่อนำไฟล์ `.xlsx` ที่ส่งออกไว้กลับเข้าระบบ
